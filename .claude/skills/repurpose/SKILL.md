# /repurpose — Extract Shorts from Long-Form

Reads an existing long-form script.json and extracts 3-5 standalone short-form video candidates, each with a rewritten hook optimized for vertical scroll-stop.

## Usage

```
/repurpose <VideoName>
/repurpose <VideoName> --count 3
```

## Flags

| Flag | Description |
|------|-------------|
| `--count <N>` | Number of clips to extract (default: 5, range: 3-5) |

## Examples

```
/repurpose HowTheWebWorks
/repurpose TheAIStackExplained --count 3
```

## Prerequisites

- `src/videos/<VideoName>/script.json` must exist (generated by `/script`)
- Works best with explainer/tutorial videos (more extractable moments)

## What happens

### Phase 1: ANALYZE LONG-FORM CONTENT

Read `src/videos/<VideoName>/script.json` and identify:

1. **Self-contained "aha" moments** — segments that make sense WITHOUT the full video context
2. **Quotable narration** — punchy statements, surprising stats, bold claims
3. **Visual-heavy scenes** — DiagramFlow, ComparisonSplit, StatHighlight, CodeDisplay work great standalone
4. **Natural breakpoints** — sections that have a clear start and end

For each candidate, extract:
- Scene range (which scenes from the original)
- Narration subset (the actual text)
- Key message (what the viewer learns in isolation)

### Phase 2: EVALUATE HOOK QUALITY

For each candidate clip, assess:

| Criterion | Rating |
|-----------|--------|
| **Standalone value** | Does it make sense without full video context? |
| **Hook potential** | Can the first 1-3 seconds stop a scroll? |
| **Visual strength** | Are the scene types visually engaging for vertical? |
| **Length fit** | Does it land in the 13s or 60s sweet spot? |

Rate each: **Strong / Medium / Weak**. Drop anything rated Weak on standalone value.

### Phase 3: REWRITE HOOKS

For each candidate, **rewrite the opening** — do NOT lift the hook verbatim from the long-form video. The short needs its own scroll-stopping opener.

Apply the youtube skill's shorts hook framework:
- Load `.agents/skills/youtube/sub-skills/shorts.md` for hook design rules
- Load `.agents/skills/youtube/references/shorts-playbook.md` for algorithm signals
- Each hook must create a curiosity gap or pattern interrupt in 1-3 seconds
- Each hook must work for someone who has NEVER seen the long-form video

### Phase 4: GENERATE SHORT-FORM SCRIPTS

For each accepted candidate, generate a complete short-form `script.json`:

```json
{
  "videoName": "<VideoName>Short<N>",
  "type": "short",
  "fps": 30,
  "resolution": { "width": 1080, "height": 1920 },
  "meta": {
    "targetDuration": 60,
    "loopSetup": "<how ending mirrors opening>",
    "hookStrategy": "<chosen archetype>",
    "shortGoal": "long-form funnel",
    "repurposedFrom": "<VideoName>",
    "sourceScenes": [3, 4, 5]
  },
  "sections": [{ ... }]
}
```

Key differences from standalone shorts:
- `meta.repurposedFrom` — links back to the source video
- `meta.sourceScenes` — which scenes from the original were extracted
- `meta.shortGoal` defaults to `"long-form funnel"` (drive views to the full video)
- Hook is rewritten (not copied from original)
- Loop setup is added (original likely doesn't have one)

### Phase 5: RANK AND PRESENT

Present all candidates as a ranked table:

| Rank | Clip | Hook Quality | Standalone | Target Duration | Key Message |
|------|------|-------------|------------|-----------------|-------------|
| 1 | `<VideoName>Short1` | Strong | Yes | 60s | [message] |
| 2 | `<VideoName>Short2` | Strong | Yes | 13s | [message] |
| ... | ... | ... | ... | ... | ... |

Include:
- **Posting order**: Which to post first (strongest hook first)
- **Spacing**: 2-3 days between posts
- **Bridge element**: How each short can funnel viewers to the long-form video (end CTA, pinned comment, etc.)

## Output

```
src/videos/<VideoName>Short1/
  script.json        ← short-form script ready for /video

src/videos/<VideoName>Short2/
  script.json

src/videos/<VideoName>Short3/
  script.json

(up to 5 clips)
```

## After reviewing

Run `/video <VideoName>Short1` to generate Remotion code for each short, then `/voiceover` and `/assets`.

## Quality Criteria

- Each clip has genuine standalone value (makes sense without full video)
- Hooks are rewritten for vertical scroll-stop (NOT copied verbatim)
- Each clip targets a sweet spot duration (13s or 60s)
- Loop setup connects final scene back to opening
- No two clips cover the same content
- Posting order is justified by hook strength
- All clips use vertical-compatible scene types
