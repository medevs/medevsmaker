# Audio Integration — Remotion Patterns

## VoiceoverLayer Component

`src/shared/components/VoiceoverLayer.tsx` renders `<Audio>` elements inside `<Sequence>` wrappers.

### Props

```tsx
type VoiceoverScene = {
  src: string;           // Path relative to public/ (e.g., "vo/HowTheWebWorks/scene-01.mp3")
  fromFrame: number;     // Absolute frame where audio starts
  durationInFrames: number;  // How long the audio plays
};

type VoiceoverLayerProps = {
  scenes: VoiceoverScene[];
  volume?: number;       // 0-1, default 1
};
```

### Integration Pattern

VoiceoverLayer sits as a sibling to the main video content in the composition's `index.tsx`:

```tsx
import { VoiceoverLayer } from "../shared/components/VoiceoverLayer";
import { VOICEOVER_SCENES } from "./voiceover";

export const MyVideo: React.FC = () => {
  return (
    <AbsoluteFill>
      <Background ... />
      <Series>{/* sections */}</Series>
      <Watermark ... />
      <VoiceoverLayer scenes={VOICEOVER_SCENES} />
    </AbsoluteFill>
  );
};
```

## voiceover.ts Module

Auto-generated by `generate-audio.ts`. Contains the `VOICEOVER_SCENES` array with absolute frame offsets.

```ts
import type { VoiceoverScene } from "../shared/components/VoiceoverLayer";

export const VOICEOVER_SCENES: VoiceoverScene[] = [
  {
    src: "vo/HowTheWebWorks/scene-01.mp3",
    fromFrame: 0,
    durationInFrames: 150,
  },
  // ...
];
```

**Do not edit manually** — regenerate by running `generate-audio.ts`.

## Audio Files

- Stored in `public/vo/<VideoName>/` (gitignored)
- Named `scene-01.mp3`, `scene-02.mp3`, etc.
- Referenced via `staticFile()` in Remotion
- MP3 format, 128kbps

## Frame Offset Calculation

Frame offsets account for:
1. **TransitionSeries overlaps** within sections — scenes start earlier by the transition's `durationInFrames`
2. **Series sequencing** between sections — each section starts after the previous one's total `durationFrames`

The `computeFrameOffsets()` utility in `scripts/tts/utils.ts` handles this calculation using the manifest data.

## Key Remotion Rules

- `<Audio>` must be inside a `<Sequence>` to control timing
- Use `staticFile()` for audio in `public/` directory
- `premountFor={30}` on audio Sequences ensures smooth playback start
- Audio is automatically mixed during render (`npx remotion render`)
- In Studio preview, audio plays in real-time at the current frame position

## Rendering with Audio

```bash
# Preview (audio plays in real-time)
npx remotion studio

# Render to MP4 (audio embedded)
npx remotion render src/index.ts <VideoName> out/<VideoName>.mp4
```

Remotion automatically mixes all `<Audio>` elements into the final render.
