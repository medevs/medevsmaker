# /voiceover — Voiceover Director

Adds natural voiceover narration to an existing video generated by `/video`.

## Usage

```
/voiceover <VideoName>
```

## Examples

```
/voiceover HowTheWebWorks
/voiceover TheAIStackExplained
/voiceover UnderstandingDatabases
```

## What happens

1. **Manifest check** — Reads or generates `manifest.json` for the video
2. **Transcript generation** — Creates a transcript skeleton, then AI writes natural narration for each scene
3. **User review** — You review and edit the transcript before synthesis
4. **TTS synthesis** — Calls Cartesia (or configured provider) to generate MP3 for each scene
5. **Audio integration** — Adds `VoiceoverLayer` to the video composition

## Prerequisites

- The video must already exist in `src/<VideoName>/` (generated by `/video`)
- `CARTESIA_API_KEY` must be set in `.env`
- `CARTESIA_VOICE_ID` must be set in `.env` (clone your voice at https://play.cartesia.ai first)

## Voice cloning

1. Record **10+ seconds** of clear audio (quiet room, good mic, natural speech)
2. Go to https://play.cartesia.ai → Voices → Clone
3. Upload your audio sample, preview and test the cloned voice
4. Copy the voice ID and add to `.env`:
   ```
   CARTESIA_VOICE_ID=<voice-id>
   ```

**Tip**: Test with a Cartesia stock voice first to verify the pipeline works, then switch to your clone.

## Environment variables

```bash
TTS_PROVIDER=cartesia              # default
CARTESIA_API_KEY=<key>             # required
CARTESIA_VOICE_ID=<voice-id>      # required — your cloned voice
CARTESIA_MODEL=sonic-2             # sonic-2 (default) | sonic-3 | sonic
CARTESIA_SPEED=slow                # slow (recommended) | normal | fast
```
