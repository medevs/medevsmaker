# Voiceover Director Skill

You are an expert Voiceover Director. This skill powers two commands in the validated pipeline:

```
/transcript <VideoName>   → Phase 4: Transcript Generation → transcript.json (user reviews)
/voiceover <VideoName>    → Phase 5: TTS Synthesis → MP3 files
                          → Phase 6: Audio Integration → VoiceoverLayer added to index.tsx
```

---

## PHASE 4: TRANSCRIPT GENERATION (triggered by `/transcript`)

**Goal**: Create natural voiceover narration for every scene, guided by the `narrationIntent` planned during `/video`.

### Prerequisites

1. Verify `src/<VideoName>/manifest.json` exists (generated by `/video` Phase 3)
2. If no manifest exists, generate one by reading the section TSX files (see Manifest Generation below)

### Steps

1. **Generate skeleton** — Run `node --env-file=.env --strip-types scripts/tts/generate-transcript.ts <VideoName>` to create `transcript.json` with empty narration fields (includes `narrationIntent` and `narratorTone` from manifest)
2. **Write narration** — Fill the `narration` field for each scene following the rules in [rules/transcript-generation.md](rules/transcript-generation.md)
   - Use `narrationIntent` as primary creative direction — expand the intent, don't write from scratch
   - Use `onScreenText` to ensure narrator never reads screen text verbatim
   - Use `narratorTone` for emotional guidance per scene
3. **Present for review** — Show the complete transcript to the user. Wait for approval before proceeding.
4. **After user approval** — Tell the user: "Run `/voiceover <VideoName>` to synthesize audio."

### Narration Writing Rules

Full rules: [rules/transcript-generation.md](rules/transcript-generation.md)

Key principles:
- **narrationIntent is your primary input** — each scene has a creative directive from the video director. Expand it into natural narration.
- **Never read on-screen text verbatim** — check `onScreenText` array and complement, don't duplicate
- **Stay at or under word budgets** — budget uses adaptive WPM (190/160/140 for short/medium/long scenes). Exceeding budget risks audible cutoff
- **Count words** — verify every narration is near budget before presenting for review
- **Always narrate SectionTitle scenes** — silent section gaps make the audio feel broken. Use a brief 4-6 word transition phrase
- **Explain, don't just label** — narration should teach concepts, not just name them. Use analogies that make abstract ideas concrete
- **Conversational peer tone** — match the video's audience profile (Fireship-style humor, 1 joke per section)
- **Prefer flowing sentences** — TTS adds pauses for periods/commas that inflate duration. Staccato fragments ("Step one. Step two.") take longer than their word count suggests
- **Scene-type patterns** — different scene types need different narration approaches

## PHASE 5: TTS SYNTHESIS (triggered by `/voiceover`)

**Goal**: Convert the approved transcript into MP3 audio files.

### Prerequisites

1. `transcript.json` must exist with narration filled for each scene (run `/transcript` first)
2. Verify narration fields are non-empty — if any scene has empty narration, warn the user and suggest running `/transcript` first
3. TTS provider configured in `.env` (default: Cartesia; requires `CARTESIA_VOICE_ID` + `CARTESIA_API_KEY`)
4. See [rules/tts-providers.md](rules/tts-providers.md) for provider setup

### Steps

1. **Check transcript** — Read `transcript.json` and verify narration fields are populated
2. **Run synthesis** — Execute `node --env-file=.env --strip-types scripts/tts/generate-audio.ts <VideoName>`
3. **Verify output** — Check that MP3 files were created in `public/vo/<VideoName>/`
4. **Check durations** — Ensure audio durations roughly match scene durations (±1 second)
5. **Report results** — Show the user a summary of generated files and durations

The script automatically:
- Generates MP3 files to `public/vo/<VideoName>/`
- Updates `transcript.json` with `audioFile` and `actualDurationSeconds`
- Creates `src/<VideoName>/voiceover.ts` with the `VOICEOVER_SCENES` array

## PHASE 6: AUDIO INTEGRATION (triggered by `/voiceover`, after Phase 5)

**Goal**: Wire the voiceover audio into the Remotion composition.

### Steps

1. **Add import** to `src/<VideoName>/index.tsx`:
   ```tsx
   import { VoiceoverLayer } from "../shared/components/VoiceoverLayer";
   import { VOICEOVER_SCENES } from "./voiceover";
   ```

2. **Add VoiceoverLayer** as sibling to `<Series>` (after Watermark):
   ```tsx
   <AbsoluteFill>
     <Background ... />
     <Series>{/* sections */}</Series>
     <Watermark ... />
     <VoiceoverLayer scenes={VOICEOVER_SCENES} />
   </AbsoluteFill>
   ```

3. **Provide rendering instructions**:
   ```bash
   # Preview with voiceover
   npx remotion studio

   # Render with audio
   npx remotion render src/index.ts <VideoName> out/<VideoName>.mp4
   ```

## MANIFEST GENERATION (Fallback)

If no `manifest.json` exists, generate one by parsing the video's section files:

1. Read `src/<VideoName>/styles.ts` for FPS, section frame constants
2. Read each `src/<VideoName>/sections/SectionN.tsx`
3. Extract scene types, durations (from `durationInFrames / FPS`), and props
4. Extract transition types and frame durations
5. Write `manifest.json` with the extracted data (set `narrationIntent`, `onScreenText`, `narratorTone` to empty defaults)

### Manifest Format

```json
{
  "videoName": "HowTheWebWorks",
  "fps": 30,
  "totalFrames": 5833,
  "meta": {
    "learningObjectives": ["Understand client-server model"],
    "audienceProfile": "vibe-coders",
    "humorStyle": "dry-tech"
  },
  "sections": [
    {
      "sectionIndex": 1,
      "title": "The Client",
      "durationFrames": 1250,
      "sectionColor": "#6366f1",
      "sectionTone": "curious, building anticipation",
      "humorScene": 5,
      "scenes": [
        {
          "sceneIndex": 1,
          "sceneType": "HookQuestion",
          "durationSeconds": 5,
          "transitionAfter": { "type": "fade", "frames": 15 },
          "narrationIntent": "Create urgency — make them feel they're missing something",
          "onScreenText": ["What actually happens when you click a link?"],
          "narratorTone": "mysterious, slightly playful",
          "props": {
            "question": "What actually happens when you click a link?",
            "subtext": "Spoiler: it's not magic, but it's close"
          }
        }
      ]
    }
  ]
}
```

## IMPORTANT CONSTRAINTS

- Always generate manifest.json before transcript.json
- Never proceed to TTS without user approval of the transcript
- Stay at or under word budgets (adaptive WPM: 190/160/140) — exceeding budget causes audio cutoff
- Use `staticFile()` for audio paths in Remotion (files in `public/`)
- VoiceoverLayer renders no visuals — it's purely audio Sequences
- SectionTitle scenes should always have brief narration (4-6 words) — silent gaps confuse viewers
- EndScreen should have a brief conversational CTA (8-12 words) — silence over end cards feels unfinished
- The voiceover.ts file is auto-generated — don't edit it manually
- `/transcript` generates transcript only — tell user to run `/voiceover` after approval
- `/voiceover` assumes transcript exists — check for non-empty narrations before TTS
